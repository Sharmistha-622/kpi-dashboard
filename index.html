Stage 1 — Discovery deep dive

What you produce

Process map (as-is) Opportunity assessment Effort estimate Benefits projection

Who is involved

Process owner Operations lead Subject matter expert

Gate 1 checklist — must pass to proceed

✓ Objective clearly defined ✓ Process mapped end-to-end ✓ Business case exists ✓ Business owner assigned

Common blockers at this stage

Process not documented No clear owner Benefits not measurable Stakeholder unavailable

Stage 2 — Feasibility assessment deep dive

What you produce

Future-state workflow ROI analysis document Tool selection report Risk register

Who is involved

Automation analyst IT / tech lead Finance (for ROI) Compliance

Gate 2 checklist — must pass to proceed

✓ ROI positive & approved ✓ Tool selected & licensed ✓ Risks assessed & mitigated ✓ Future workflow signed off

Common blockers at this stage

ROI difficult to quantify Tool licensing delays Data access issues Approval bottleneck

Stage 3 — Build, test & validate deep dive

What you produce

Working automation Test results (Unit/E2E/UAT) Exception handling docs Rollback plan

Who is involved

Developer / RPA engineer QA tester Business UAT team Security reviewer

Gate 3 checklist — must pass to proceed

✓ All tests passed (Unit/E2E/UAT) ✓ Exception handling validated ✓ Rollback plan documented ✓ Security sign-off obtained

Common blockers at this stage

UAT stakeholder unavailable Integration failures Changing requirements Missing test environment

Stage 4 — Deploy, monitor & optimise deep dive

What you produce

Live automation in production KPI monitoring dashboard Periodic review report Optimisation log

Who is involved

Operations team Business owner IT support CEO / leadership

Success KPIs to track

Time saved per week Error rate reduction Cost savings (Rs) ROI achieved vs projected Process uptime %

Common blockers at this stage

No monitoring set up KPI not measured post-launch No owner after go-live No improvement process
